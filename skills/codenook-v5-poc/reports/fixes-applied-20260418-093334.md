## Bug 5 — "no skipping subtask phases" enforcement

**Files**
- `templates/CLAUDE.md`
- `templates/agents/planner.agent.md`
- `templates/core/codenook-core.md` (§17.3)
- `templates/dispatch-audit.sh` (check [8])

**Change**
- CLAUDE.md Step 1 hard-rules list grew a "Subtask pipelines are non-skippable" bullet pointing at §17.3 + audit check [8].
- Planner agent profile gained a "Subtask Pipeline Rule (MANDATORY)" section codifying the same rule from the planner's perspective (since the planner authors `start_phase`).
- core.md §17.3 grew a "No skipping subtask phases (Bug §5)" subsection.
- New `dispatch-audit.sh` check `[8]`: for every subtask whose `state.json` status is not `pending`, verifies the dispatch log includes every phase in the pipeline starting from `start_phase` (default `clarify`). Phase-from-log matching tolerates the existing manifest naming (`phase-N-<role>`, `iter-N-implementer`, etc.) by mapping role names back to phase names.

**Judgement calls**
- Audit check [8] looks at the pipeline `clarify → design → plan → implement → test → accept → validate`. `review` is not a separate phase in this pipeline (it's part of the implement loop), so it's not required.
- Subtask-status check skips `pending` only — the moment a subtask starts work, the audit becomes active. This avoids false positives on freshly-seeded but not-yet-worked subtasks.
